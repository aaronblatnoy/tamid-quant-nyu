# Phase 02 — Sea-route distance

## Metadata
- Effort: `max`
- Depends on phases: 00
- Applies security-review: `no`
- Max phase runtime (minutes): 120
- External services: none

## Mission
Tightness is ton-miles forward, where "miles" must be **sea-route** miles
(actual sailing distance through navigable water), not great-circle.
Great-circle PG→China routes pass over land and short-change the true
ton-mile demand by ~8–10%. That bias would systematically understate
tightness when it matters most (when Malacca is congested and Sunda
diversions add hundreds of miles). This phase integrates `searoute-py`,
builds a pair-keyed distance cache parquet, and snapshot-tests three
known sea-route distances against industry reference values so regressions
in the underlying library show up immediately.

## Acceptance criteria
- `packages/ais/src/taquantgeo_ais/gfw/distance.py` exists with:
  - `compute_route_distance(origin_lat_lon, dest_lat_lon, *, prefer_malacca:
    bool = True) -> float` — nautical miles
  - `build_distance_cache(anchorage_pairs, out_path) -> polars.DataFrame`
  - Module docstring explains Malacca-preferred routing choice and the
    fallback behavior on disconnected components (great-circle + WARN log)
- `data/processed/distance_cache.parquet` after `uv run taq gfw
  compute-distances --voyages-dir data/processed/voyages --out
  data/processed/distance_cache.parquet` against test fixture.
  Schema: `origin_s2id` (str), `dest_s2id` (str), `origin_lat`, `origin_lon`,
  `dest_lat`, `dest_lon`, `nautical_miles` (float64),
  `is_great_circle_fallback` (bool), `computed_at` (timestamp[us, UTC]).
  Primary key: `(origin_s2id, dest_s2id)`.
- CLI: `taq gfw compute-distances --help` exits 0. Behavior: loads voyages
  parquet, collects unique (start_anchorage_id, end_anchorage_id) pairs,
  joins to anchorages for lat/lon, computes distance for each pair,
  writes cache. Re-runs skip pairs already in cache unless `--force`.
- Snapshot tests pinning these distances (tolerance ±3% to absorb
  searoute waypoint updates):
  - Ras Tanura (26.70, 50.18) → Ningbo (29.87, 121.55) via Malacca:
    ≈ 6,250–6,400 NM
  - Ras Tanura → Ningbo via Sunda (force Malacca off): ≈ 6,800–6,950 NM
  - Basrah OT (29.72, 48.83) → Qingdao (36.07, 120.30) via Malacca:
    ≈ 6,400–6,600 NM
  Tests document (in docstring) WHY each pinned number is correct by
  citing either Worldscale flat-rate data or the `searoute-py` reference
  dataset version used.
- `test_distance_disconnected_returns_great_circle_with_warn` — passing
  two points on opposite sides of a land mass with no sea route between
  them returns a great-circle fallback and the returned row has
  `is_great_circle_fallback=True`.
- `test_distance_idempotent_cache` — second CLI invocation does not
  re-compute cached pairs.
- All quality gates green.

## Non-goals
- Live routing for in-progress voyages (next position → destination). The
  cache is (anchorage → anchorage); in-progress voyage ton-mile-remaining
  is computed per-vessel by phase 04 using current AIS position + this
  cache as the reference distance.
- Canal/strait congestion dynamics. Static routing only. (Candidate entry
  for future work.)
- Dark-fleet routing (AIS-off legs). Out of scope — phase 03 territory.
- Seasonal ice routing (Northern Sea Route). Not relevant to TD3C.
