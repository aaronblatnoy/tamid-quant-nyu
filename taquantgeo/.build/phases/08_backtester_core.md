# Phase 08 — Backtester core

## Metadata
- Effort: `max`
- Depends on phases: 04, 06, 07
- Applies security-review: `no`
- Max phase runtime (minutes): 180
- External services: none (reads Postgres signals + prices into memory)

## Mission
Given phase 07 confirmed edge, phase 08 builds the backtester that turns
the signal into a trading result we can measure. Long-or-flat equity
basket, entered on tightness z-score crossings, exited on the opposite
crossing or a max-hold timer. T+1 execution STRICTLY enforced: signal
computed at close of day D triggers trades at the open of day D+1. No
intraday. No look-ahead. Deterministic for a fixed input — seed any
random choices and document. Output is an equity curve parquet + trades
parquet + stats dict the walk-forward phase consumes. No portfolio
optimization, no transaction-cost models beyond a flat slippage — those
are candidate entries.

## Acceptance criteria
- NEW package at `packages/backtest/` registered in workspace.
- `packages/backtest/src/taquantgeo_backtest/engine.py` with:
  - `@dataclass(frozen=True) class BacktestConfig: start_date; end_date;
     tickers; entry_z; exit_z; max_hold_days; equity_per_name_usd;
     max_gross_usd; slippage_bps; seed`
  - `@dataclass(frozen=True) class BacktestResult: trades_df;
     equity_curve_df; stats: dict[str, float]`
  - `run_backtest(signals_df, prices_df, config) -> BacktestResult`
- T+1 enforcement: any signal observed at `as_of = D` can only drive a
  trade at open of day `D+1` (first subsequent trading day). Unit test
  verifies this with a hand-crafted series.
- Stats dict contains: `sharpe`, `sortino`, `calmar`, `max_drawdown`,
  `hit_rate`, `avg_holding_period_days`, `turnover_annual`, `vol_annual`,
  `total_return`, `cagr`, `n_trades`.
- Output parquets:
  - `trades.parquet`: `ticker`, `entry_ts`, `exit_ts`, `entry_price_cents`,
    `exit_price_cents`, `qty`, `pnl_cents`, `pnl_bps`, `hold_days`,
    `entry_signal_z`, `exit_signal_z`, `exit_reason`
  - `equity_curve.parquet`: `ts` (date), `equity_cents`,
    `drawdown_cents`, `n_open_positions`, `gross_exposure_cents`
- Deterministic: two runs with the same `BacktestConfig` produce
  identical bytes for both parquet files. Test: SHA256 compare two
  outputs.
- CLI: `taq backtest run --config backtest/v1_config.json [--out
  backtest_results/v1/]` (backtest_results/ is gitignored).
- Tests:
  - `test_t_plus_one_execution_never_uses_same_day_open`
  - `test_equal_weight_basket_caps_per_name_usd`
  - `test_max_hold_exit`
  - `test_z_score_entry_exit_crossings`
  - `test_slippage_reduces_pnl_by_configured_bps`
  - `test_kill_switch_config_flag_halts_new_entries` — if config flag
    set, no new entries after that date, but existing positions can exit
    (simulation of the operational kill-switch path)
  - `test_determinism_across_two_runs`
  - `test_stats_sharpe_matches_hand_calc` — synthetic equity curve with
    known Sharpe; compare
  - `test_max_drawdown_from_peak` — synthetic curve with one known DD
  - `test_empty_signals_produces_empty_results_not_crash`
  - `test_missing_ticker_prices_skips_that_ticker_logs_warn`
- All quality gates green.

## Non-goals
- Long/short — v0 is long/flat. Shorting shipping equities carries
  borrow-cost considerations we're not modeling. Candidate entry.
- Transaction cost beyond flat slippage — no bid/ask, no market impact,
  no commission schedule. v0 simplification. Candidate entry.
- Intraday data — daily bars only. TD3C signal is daily.
- Multi-horizon ensemble — one horizon selected per config.
